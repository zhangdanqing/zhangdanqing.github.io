NocoBase User Guide ​

NocoBase is a powerful open-source low-code platform designed to provide users with highly flexible solutions for application development and data management. With its visual drag-and-drop interface and plugin-based architecture, users can quickly build custom applications without requiring extensive programming knowledge. This article will cover NocoBase's core features, source code analysis, and usage methods to help you understand and efficiently use NocoBase.

Core Features Overview ​

NocoBase provides a comprehensive set of core features to simplify development. The key modules include:

1. Data Management ​

NocoBase's data management features are based on relational database design. Users can directly create, modify, and manage data tables and their relationships through the interface. Key functionalities include:

  • Table Structure Design: Supports defining field types (text, number, date, JSON, etc.).
  • Data Operations: Supports CRUD operations.
  • Relationships: Supports one-to-one, one-to-many, and many-to-many relationships.

2. Permission Management ​

With a flexible permission management module, users can define rules for:

  • Access control for different user roles.
  • Data-level permissions (e.g., row-level permissions).
  • Field-level permissions.

3. Workflow Automation ​

NocoBase provides a powerful workflow design tool that supports event-driven automation, such as:

  • Triggering notifications after data creation.
  • Executing scheduled tasks.
  • Handling conditional branching.

4. Plugin Architecture ​

One of NocoBase's core designs is its plugin architecture, allowing functionalities to be extended without modifying the core code. The community currently offers a rich library of plugins, such as:

  • File Management Plugin
  • User Authentication Plugin
  • Data Analysis Plugin

5. Customizable Frontend Interfaces ​

With the drag-and-drop designer, users can customize frontend interfaces to quickly generate interactive admin panels or user-facing pages.

Source Code Structure Analysis ​

NocoBase has a clear source code structure, mainly divided as follows:

plaintext
NocoBase/
├── packages/          # Core modules and plugins
│   ├── core/          # Core functionality implementation
│   ├── plugins/       # Plugin features
│   ├── database/      # Database-related modules
│   └── utils/         # Common utility functions
├── examples/          # Example projects
├── scripts/           # Build and deployment scripts
├── docs/              # Documentation resources
└── tests/             # Unit and integration tests

Core Modules ​

  1. core/: Implements NocoBase's core framework logic, including startup processes and plugin loading mechanisms.
  2. database/: Abstract layer for database operations, supporting multiple databases (default support for SQLite and MySQL).
  3. plugins/: Built-in basic plugins, such as user authentication and file management.

Plugin Mechanism ​

Plugins are modularly designed, allowing developers to quickly create custom plugins using @nocobase/plugin-xxx and load them into the system via configuration files.

Installation and Quick Start ​

Here are the steps to install and quickly get started with NocoBase:

1. Install NocoBase ​

Prerequisites ​

  • Node.js >= 14.0.0
  • npm >= 6.0.0 or Yarn
  • Database (MySQL or SQLite)

Installation Steps ​

bash
# Clone the repository
git clone https://github.com/nocobase/nocobase.git

# Enter the project directory
cd nocobase

# Install dependencies
npm install

# Start the service
npm run dev

Once the service starts, access http://localhost:13000 to enter the admin dashboard.

2. Configure the Database ​

By default, SQLite is used. You can switch to MySQL by modifying the configuration file:

In config/database.js:

javascript
module.exports = {
  dialect: 'mysql',
  host: 'localhost',
  username: 'root',
  password: 'password',
  database: 'nocobase',
};

Basic Usage ​

Creating Data Tables ​

  1. Log in to the admin dashboard and go to "Model Management."
  2. Click "Create Table."
  3. Define the table name and fields:
    • Field types include text, number, date, boolean, etc.
    • You can set whether fields are required or unique.

Defining Model Fields ​

When creating data tables, you can add validation rules and default values for fields. For example:

  • String fields can have length limits.
  • Numeric fields can have minimum and maximum values.

Configuring Permissions ​

  1. Go to the "Permission Management" module.
  2. Create roles and assign permissions to them:
    • Data access permissions: Specify which tables or fields can be accessed.
    • Operational permissions: Specify which actions (e.g., create, delete, update) are allowed.

Designing Workflows ​

  1. Enter the "Workflow Designer."
  2. Add event triggers, such as "When data is created."
  3. Add task nodes, such as "Send Email" or "Call API."
  4. Configure conditional branching logic.

Advanced Features ​

Plugin System ​

NocoBase's plugins can extend the platform's functionality. Here are the steps to install and use plugins:

Installing Plugins ​

bash
npm install @nocobase/plugin-xxx

Loading Plugins ​

Add the plugin to config/plugins.js:

javascript
module.exports = [
  '@nocobase/plugin-xxx',
];

Common Plugins ​

  • @nocobase/plugin-file-manager: File upload and management.
  • @nocobase/plugin-charts: Data visualization.

Customizing Interfaces ​

Using the drag-and-drop interface designer, you can:

  1. Customize form layouts.
  2. Create tailored charts and data displays.
  3. Define buttons and interaction logic.

Integrating Third-Party Services ​

Through REST API or Webhooks, NocoBase can integrate with external systems. For example:

  • Send HTTP requests to external services.
  • Receive Webhook requests from third-party systems.

Best Practices and Case Studies ​

  1. Small Business Management System

    • Features: Order management, customer management, inventory management.
    • Implementation: Combines NocoBase's table relationships and permission management modules.

  2. Education and Training Institution Management System

    • Features: Course scheduling, student management, grade tracking.
    • Implementation: Automated notifications via workflows.

  3. Data Visualization Platform

    • Features: Real-time business data display.
    • Implementation: Uses data analysis plugins and custom chart interfaces.

With this guide, you should be able to quickly get started with NocoBase and build efficient applications. For deeper insights into its source code or extended features, refer to the NocoBase Official Documentation.
