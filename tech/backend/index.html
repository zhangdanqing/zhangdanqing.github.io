使用 NestJS 搭建后端项目 ​

NestJS 是一个用于构建高效、可靠和可扩展的服务端应用程序的渐进式 Node.js 框架。其模块化架构和对 TypeScript 的广泛使用，使其成为现代后端开发的理想选择。本指南提供了分步骤的教程，帮助您使用 NestJS 搭建一个后端管理系统，涵盖从安装到高级配置的所有内容。

NestJS 简介 ​

NestJS 基于 Express（或可选的 Fastify）构建，默认使用 TypeScript，提供强类型和现代编程范式。它采用了类似 Angular 的模块化架构，使代码高度组织化且易于扩展。

为什么选择 NestJS？ ​

  • TypeScript 支持：完全支持 TypeScript，包括装饰器和元数据。
  • 模块化架构：便于组织和扩展应用。
  • 内置功能：包括依赖注入、认证和数据库集成的解决方案。
  • 广泛的生态系统：支持 WebSockets、GraphQL、REST API、微服务等。

搭建 NestJS 项目 ​

前置要求 ​

确保您已安装以下工具：

  • Node.js (>= 16.x)
  • npm 或 yarn

安装步骤 ​

  1. 全局安装 NestJS CLI：
bash
npm install -g @nestjs/cli
  1. 创建新项目：
bash
nest new my-backend-project
  1. 进入项目目录：
bash
cd my-backend-project
  1. 启动开发服务器：
bash
npm run start:dev

访问 http://localhost:3000 查看运行中的应用。

项目结构概览 ​

NestJS 遵循模块化结构，保持代码库的组织性和可扩展性。以下是默认结构的简要说明：

plaintext
src/
├── app.controller.ts       # 主控制器
├── app.module.ts           # 根模块
├── app.service.ts          # 主服务
├── main.ts                 # 应用程序入口
├── modules/                # 自定义模块
│   ├── users/              # 示例模块
│   │   ├── users.controller.ts
│   │   ├── users.module.ts
│   │   ├── users.service.ts
  • 模块：组织应用功能。
  • 控制器：定义端点并处理请求。
  • 服务：包含业务逻辑并与数据库交互。

创建模块、控制器和服务 ​

第一步：生成模块 ​

使用 CLI 生成新模块：

bash
nest generate module users

这将在 src/modules 目录下创建一个 users 模块文件。

第二步：生成控制器 ​

bash
nest generate controller users

users.controller.ts 文件将处理传入的 HTTP 请求并将其路由到相应的服务方法。

第三步：生成服务 ​

bash
nest generate service users

users.service.ts 文件将包含 users 模块的业务逻辑。

连接数据库 ​

NestJS 可以无缝集成流行的 ORM，如 TypeORM、Sequelize、Prisma 和 Mongoose。

使用 TypeORM ​

第一步：安装 TypeORM 和数据库驱动 ​

对于 MySQL：

bash
npm install --save @nestjs/typeorm typeorm mysql2

第二步：配置 TypeORM ​

编辑 app.module.ts 以包含 TypeORM 配置：

typescript
import { Module } from '@nestjs/common';
import { TypeOrmModule } from '@nestjs/typeorm';

@Module({
  imports: [
    TypeOrmModule.forRoot({
      type: 'mysql',
      host: 'localhost',
      port: 3306,
      username: 'root',
      password: 'password',
      database: 'my_database',
      autoLoadEntities: true,
      synchronize: true, // 生产环境请设置为 false
    }),
  ],
})
export class AppModule {}

第三步：定义实体 ​

为 User 表创建一个实体：

typescript
import { Entity, Column, PrimaryGeneratedColumn } from 'typeorm';

@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  name: string;

  @Column()
  email: string;
}

第四步：集成 Repository ​

将 Repository 注入到 users.service.ts 中：

typescript
import { Injectable } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { User } from './user.entity';

@Injectable()
export class UsersService {
  constructor(
    @InjectRepository(User)
    private readonly userRepository: Repository<User>,
  ) {}

  async findAll(): Promise<User[]> {
    return this.userRepository.find();
  }
}

认证和授权 ​

安装所需包 ​

bash
npm install --save @nestjs/passport passport passport-local
npm install --save-dev @types/passport-local

实现认证模块 ​

  1. 创建 auth 模块：
bash
nest generate module auth

  1. 使用 Passport.js 创建策略和守卫以保护路由。

  2. 将策略注入到应用的认证工作流中。

API 开发与验证 ​

NestJS 使用 class-validator 和 class-transformer 库内置验证功能。

第一步：安装验证库 ​

bash
npm install --save class-validator class-transformer

第二步：应用验证 ​

定义带有验证规则的 DTO（数据传输对象）：

typescript
import { IsString, IsEmail } from 'class-validator';

export class CreateUserDto {
  @IsString()
  name: string;

  @IsEmail()
  email: string;
}

在控制器中应用 DTO：

typescript
@Post()
create(@Body() createUserDto: CreateUserDto) {
  return this.usersService.create(createUserDto);
}

实现中间件和守卫 ​

中间件 ​

中间件用于在请求到达控制器之前处理请求。例如：

typescript
import { Injectable, NestMiddleware } from '@nestjs/common';

@Injectable()
export class LoggerMiddleware implements NestMiddleware {
  use(req: any, res: any, next: () => void) {
    console.log(`Request...`);
    next();
  }
}

守卫 ​

守卫决定请求是否可以继续到路由处理程序。例如：

typescript
import { Injectable, CanActivate, ExecutionContext } from '@nestjs/common';

@Injectable()
export class AuthGuard implements CanActivate {
  canActivate(context: ExecutionContext): boolean {
    const request = context.switchToHttp().getRequest();
    return request.headers.authorization === 'my-secret-token';
  }
}

部署应用 ​

使用 Docker 部署 ​

  1. 创建一个 Dockerfile：
dockerfile
FROM node:16
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
CMD ["npm", "run", "start:prod"]
  1. 构建并运行 Docker 镜像：
bash
docker build -t my-nest-app .
docker run -p 3000:3000 my-nest-app

最佳实践 ​

  • 使用 DTO 进行数据验证和转换。
  • 在生产环境中避免使用 synchronize: true。
  • 使用环境变量管理敏感配置。
  • 使用 @nestjs/common 的 Logger 类实现适当的日志记录。
  • 为控制器和服务编写单元测试。

本指南涵盖了使用 NestJS 创建后端管理系统的基础步骤。如需进一步了解，请参考 NestJS 官方文档。
